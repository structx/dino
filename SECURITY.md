# Security Policy

## Supported Versions

We currently support security updates for the following versions of Dino:

| Version | Supported          |
| ------- | ------------------ |
| v0.1.x  | :white_check_mark: |
| < v0.1  | :x:                |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

If you discover a security hole, please report it privately to ensure the safety of all Dino users.

### How to Report
- **Email:** Send your report to [security@structx.io](mailto:security@structx.io).
- **Include:** A brief description, steps to reproduce (PoC), and the potential impact.

### Our Process
1. **Acknowledgement:** You will receive a response within 48 hours.
2. **Evaluation:** We will verify the vulnerability and determine its severity.
3. **Fix:** If valid, we will prepare a patch.
4. **Disclosure:** We will release a new Docker image/binary and credit you for the discovery.

---
*Note: This policy is subject to change as the project matures.*
